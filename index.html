Bonus Features

CSV import · Crowdsourcing · Moderation workflow

📥 CSV Bulk Import

  • Parse CSV with csv-parse streaming to handle large files
  • Geocode each postcode via Postcodes.io free API (batch up to 100)
  • Deduplication check on (name + postcode) before insert
  • Use COPY FROM STDIN for 10× faster bulk inserts
  • Return import report: inserted / skipped / errors

🤝 Crowdsource Submissions

  • Public form — name, address, postcode, website, description, tags
  • Geocode postcode on submit, store location point
  • Set status = 'pending' — never shown publicly until approved
  • Email submitter with tracking link (Resend/SendGrid)
  • Rate limit: 3 submissions per IP per day

🛡 Moderation Panel

  • Protected admin route via NextAuth (GitHub/Google OAuth)
  • Queue of pending shops with map preview
  • One-click approve → status becomes 'active', revalidates ISR page
  • Reject with reason → email sent to submitter
  • Bulk-approve filtered by date / region

Run Locally

    
Get the full stack running in ~10 minutes.


    

      
1

      

        
Prerequisites

        
    
          
  • Node.js 20+ installed
    • 
          
  • PostgreSQL 15+ with PostGIS extension
    • 
          
  • Git installed
    • 
          
  • (Optional) Redis for caching
    • 
        

      

    


    

      
2

      

        
Clone & Install

        

git clone https://github.com/your-org/zero-waste-near-me

cd zero-waste-near-me


# Install backend deps

cd backend && npm install


# Install frontend deps

cd ../frontend && npm install
        

      

    


    

      
3

      

        
Set Up Environment Variables

        
Copy .env.example to .env in both /backend and /frontend:

        

          
VariableDescription

          
            
DATABASE_URLPostgreSQL connection string

            
REDIS_URLRedis URL (optional)

            
NEXT_PUBLIC_API_URLBackend API base URL

            
NEXTAUTH_SECRETRandom string for session signing

            
ADMIN_GITHUB_IDGitHub OAuth App client ID

            
ADMIN_GITHUB_SECRETGitHub OAuth App client secret

            
RESEND_API_KEYEmail API key (optional)

          
        

      

    


    

      
4

      

        
Create the Database

        

# Create database and enable PostGIS

createdb zerowaste

psql -d zerowaste -c "CREATE EXTENSION postgis;"


# Run the schema

psql -d zerowaste -f backend/db/schema.sql


# Seed with example shops

node backend/db/seed.js
        

      

    


    

      
5

      

        
Start Dev Servers

        

# Terminal 1 — Backend (port 3001)

cd backend && npm run dev


# Terminal 2 — Frontend (port 3000)

cd frontend && npm run dev
        

        
Open http://localhost:3000 — you should see the homepage with the map and sample shops!

      

    


    

      
6

      

        
Import a CSV of Shops

        
CSV format: name, address, postcode, website, description, tags

        

curl -X POST http://localhost:3001/api/admin/import \

  -H "Authorization: Bearer $ADMIN_TOKEN" \

  -F "file=@shops.csv"
        

      

    


    

      
7

      

        
Deploy to Production

        
    
          
  • Frontend → Vercel (zero config, ISR built in)
    • 
          
  • Backend → Railway or Render (free tier available)
    • 
          
  • Database → Supabase or Railway PostgreSQL with PostGIS
    • 
          
  • Add custom domain + SSL (automatic on Vercel)
    • 
          
  • Set all env vars in Vercel dashboard
